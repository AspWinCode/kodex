# Codex — AI Generation Architecture

---

## Назначение

Как Studio генерирует черновики дел, и почему сейчас это честно детерминированный шаблонный генератор, а не настоящий вызов LLM.

---

## 1. Gateway / Provider

`services/content-api/ai/gateway.mjs` экспортирует `generateDraftCase({ topic, existingIds })` и `currentProviderName()`. Gateway не знает, какой провайдер стоит за ним — это архитектурный шов для подключения настоящей LLM без изменения вызывающего кода (`server.mjs`, `POST /api/generate-draft`).

Провайдер по умолчанию — `services/content-api/ai/providers/template-provider.mjs`: набор шаблонов по темам (`if`, `while`, `loop`, `dict`, `multi-return`, `generic`), детерминированно собирает валидный (по `validateCase`) черновик дела с полями `[ЗАПОЛНИТЕ]` вместо творческого текста.

**Явно НЕ настоящий ИИ.** `currentProviderName()` возвращает `'template'`, это значение показывается методисту в Studio (`Черновик готов (провайдер: template)`) — никогда не выдаётся за LLM-генерацию.

---

## 2. Как подключить настоящий LLM-провайдер (не сделано, путь описан)

1. Новый файл `services/content-api/ai/providers/<name>-provider.mjs` с той же сигнатурой `generateDraftCase({ topic, existingIds })`, возвращающей объект дела, проходящий `validateCase`.
2. Переменная окружения (например `AI_PROVIDER=openai`, `OPENAI_API_KEY=...`) — выбор провайдера в `gateway.mjs` по env, без хардкода.
3. Черновик от настоящего провайдера проходит **тот же** гейт качества и **тот же** путь публикации (`PUT /api/content/:id` → draft → review → approved), что и любая ручная правка — AI не получает привилегированный путь записи (это уже гарантировано текущей архитектурой, менять не нужно).
4. Ключ доступа — решение и стоимость на усмотрение владельца проекта, не принимается автоматически.

---

## 3. Что уже проверено на реальном производственном цикле

`CASE-011` прошёл весь путь: AI-черновик (шаблонный) → правка методистом в Studio → игра и проверка в Player → перенос в постоянный сид `data.js`. Это подтверждает, что конвейер публикации работает независимо от того, какой Provider стоит за Gateway.
