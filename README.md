# Kodex / Codex

Образовательная игровая платформа: обучение программированию под маской работы в детективном агентстве «Кодэкс».

## Структура репозитория

```
kodex/
├── apps/
│   ├── player/           — Codex Player: игровой клиент агента
│   └── studio/            — Codex Studio: минимальный редактор дел для методистов
├── docs/                  — архитектурная документация платформы (источник истины)
├── packages/
│   └── game-data/          — общие данные мира (дела, кураторы, ранги) + слияние Studio-правок,
│                              читается и Player, и Studio
├── scripts/                 — служебные скрипты (Quality Gate для CI)
└── .github/workflows/        — CI
```

## Запуск локально

```bash
python -m http.server 4173
```

Сервер обязан раздавать **корень репозитория**, а не `apps/player` — Player и Studio работают на одном origin (`http://localhost:4173`), потому что делят `localStorage`: правки, сделанные в Studio, должны быть видны Player без бэкенда.

- Codex Player: `http://localhost:4173/apps/player/` (или просто `http://localhost:4173/` — корневой `index.html` редиректит)
- Codex Studio: `http://localhost:4173/apps/studio/`

### Кэш при локальной разработке

`python -m http.server` не отправляет заголовки управления кэшем, поэтому браузер может закэшировать старую версию JS/CSS даже после `location.reload()`. Скрипты и стили подключены с версией (`?v=3`) — при следующем ощутимом изменении бампните версию в соответствующем `index.html` или используйте жёсткую перезагрузку (Ctrl+Shift+R).

## Codex Player

Одностраничное приложение без сборки — статические `HTML/CSS/JS`.

| Файл | Назначение |
|---|---|
| `apps/player/js/app.js` | Каркас, SSO-bootstrap, маршрутизация |
| `apps/player/js/state.js` | Состояние агента, персистентность (localStorage), мутации прогресса |
| `apps/player/js/engine.js` | Формализация Engine Architecture поверх state.js: именованные состояния Case/Task, журнал игровых событий (`S.events`) |
| `apps/player/js/content-overrides.js` | Накладывает Studio-правки (localStorage) на сид `CASES` при загрузке |
| `apps/player/js/screens.js` | Рендер экранов (хаб, картотека, кабинет расследования, онбординг) |
| `apps/player/js/overlays.js` | Сквозные слои: канал связи, Джарвисмен |
| `apps/player/js/ui.js` | Примитивы интерфейса, авточекер решений |

### Вход

Регистрации и авторизации внутри приложения нет — вход выполняется через SSO (токены во фрагменте URL: `#access_token=...&refresh_token=...&user=<base64url>`). Без параметров SSO в проде — редирект на внешний портал; в текущем прототипе создаётся сессия напрямую для локальной разработки.

## Codex Studio

Первая версия Studio Architecture (`docs/09`) — без бэкенда, без сборки. Один экран списка дел + форма редактирования одного дела поверх тех же данных, что читает Player.

- Список дел с пометкой статуса: **сид** (из `packages/game-data/data.js`, не тронуто), **изменено** (Studio-правка поверх сида), **новое** (создано в Studio, в сиде не существует).
- Форма редактирования: простые поля структурированы (название, куратор, награды, стартовый код и т.д.), сложные вложенные (улики/тесты, подсказки, брифинг, версии отчёта, финал) — как JSON-текст с валидацией при сохранении.
- **Гейт качества при сохранении** (зеркалирует инварианты `scripts/check.mjs`): дело не сохранится без функции, тестов у каждой улики, подсказок, брифинга, финала и хотя бы одной верной версии отчёта.
- «Сбросить» — удаляет Studio-правку, дело возвращается к сиду.
- «Экспорт правок» — показывает JSON всех Studio-правок для ручного переноса в `packages/game-data/data.js` (это и есть шаг Draft → Published при отсутствии бэкенда, см. `docs/08`).

Правки хранятся в localStorage браузера (`kodex-studio-content-v1`) и применяются к Player только на том же origin — это осознанное ограничение текущей версии, а не бэкенд.

## Дорожная карта

Проект следует поэтапному плану (`docs/16-roadmap.md`): Foundation → Core Platform → MVP → v0.2 → v0.3 (Studio) → v0.4 (AI) → v0.5 (Analytics) → Beta → Release v1.0.

Текущий этап — **v0.3 (минимальный Studio)** поверх уже реализованных Foundation / Core Platform / MVP / v0.2 (10 играбельных дел, диалоги, подсказки, Джарвисмен, журнал событий).

## Проверка качества

```bash
node scripts/check.mjs
```

Проверяет синтаксис всех JS-файлов Player, Studio и общих пакетов, а также базовые инварианты игровых данных (у каждого играбельного дела есть задача, стартовый код, улики с тестами, подсказки, брифинг, финал и хотя бы одна верная версия отчёта). Та же логика инвариантов встроена в форму сохранения Studio. Прогоняется автоматически в CI на каждый push и pull request.
