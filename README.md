# Kodex / Codex

Образовательная игровая платформа: обучение программированию под маской работы в детективном агентстве «Кодэкс».

## Структура репозитория

```
kodex/
├── apps/
│   └── player/          — Codex Player: игровой клиент агента (текущая реализация)
├── docs/                 — архитектурная документация платформы (источник истины)
├── packages/             — переиспускаемый код между приложениями (пока пуст, см. packages/README.md)
├── scripts/               — служебные скрипты (Quality Gate для CI)
└── .github/workflows/     — CI
```

## Codex Player

Одностраничное приложение без сборки — статические `HTML/CSS/JS`. Ключевые модули:

| Файл | Назначение |
|---|---|
| `apps/player/js/app.js` | Каркас, SSO-bootstrap, маршрутизация |
| `apps/player/js/state.js` | Состояние агента, персистентность (localStorage), мутации прогресса |
| `apps/player/js/engine.js` | Формализация Engine Architecture поверх state.js: именованные состояния Case/Task, журнал игровых событий (`S.events`) |
| `apps/player/js/data.js` | Данные мира: дела, кураторы, ранги, награды, учения Полигона |
| `apps/player/js/screens.js` | Рендер экранов (хаб, картотека, кабинет расследования, онбординг) |
| `apps/player/js/overlays.js` | Сквозные слои: канал связи, Джарвисмен |
| `apps/player/js/ui.js` | Примитивы интерфейса, авточекер решений |

### Кэш при локальной разработке

`python -m http.server` не отправляет заголовки управления кэшем, поэтому браузер может закэшировать старую версию JS/CSS даже после `location.reload()`. Скрипты и стили подключены с версией (`?v=2`) — при следующем ощутимом изменении бампните версию в `index.html` или используйте жёсткую перезагрузку (Ctrl+Shift+R).

### Запуск локально

```bash
python -m http.server 4173 --directory apps/player
```

Открыть `http://localhost:4173`.

### Вход

Регистрации и авторизации внутри приложения нет — вход выполняется через SSO (токены во фрагменте URL: `#access_token=...&refresh_token=...&user=<base64url>`). Без параметров SSO в проде — редирект на внешний портал; в текущем прототипе создаётся сессия напрямую для локальной разработки.

## Дорожная карта

Проект следует поэтапному плану (`docs/16-roadmap.md`): Foundation → Core Platform → MVP → v0.2 → v0.3 (Studio) → v0.4 (AI) → v0.5 (Analytics) → Beta → Release v1.0.

Текущий этап — **Foundation** (структура монорепозитория, CI) поверх уже реализованных Core Platform / MVP / части v0.2 (10 играбельных дел, диалоги, подсказки, Джарвисмен).

## Проверка качества

```bash
node scripts/check.mjs
```

Проверяет синтаксис всех JS-файлов Player и базовые инварианты игровых данных (у каждого играбельного дела есть задача, стартовый код, улики с тестами, подсказки, брифинг, финал и хотя бы одна верная версия отчёта). Прогоняется автоматически в CI на каждый push и pull request.
