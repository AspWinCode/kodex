# packages/

Переиспользуемый код без собственной точки входа/деплоя (Backend Architecture, раздел 2).

Пока пусто — появится по мере роста количества приложений (Studio, Backend), когда возникнет реальное дублирование, требующее вынесения:

- `domain-types` — общие типы предметной модели (Season/Case/Mission/Task и т.д. из Domain Model), общий язык между Player, Studio и Backend.
- `ui-kit` — общие токены дизайн-системы и примитивы интерфейса, если Studio будет переиспользовать визуальный язык Player.
- `validation` — общие проверки инвариантов Domain Model.

Правило (Engineering Handbook, YAGNI): пакет создаётся не заранее «на будущее», а в момент, когда появляется второе реальное место использования одного и того же кода.
